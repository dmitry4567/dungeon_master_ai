## Полнота требований

- [ ] CHK001 - Определены ли требования к визуальному дизайну фэнтези/средневековой темы с конкретными цветовыми палитрами или стилевыми референсами? [Пробел, Spec FR-044]
- [ ] CHK002 - Специфицированы ли скелетоны загрузки для ВСЕХ экранов с асинхронными данными (а не просто упомянуты обобщённо)? [Полнота, Spec FR-045]
- [ ] CHK003 - Указаны ли паттерны тактильной обратной связи (лёгкая, средняя, сильная) для бросков кубиков и ключевых взаимодействий? [Ясность, Spec FR-046]
- [ ] CHK004 - Определены ли UI требования для пустых состояний: список персонажей, список сценариев, история игр, список комнат? [Пробел]
- [ ] CHK005 - Специфицированы ли UI требования к сообщениям об ошибках для всех режимов сбоя аутентификации (FR-007)? [Полнота, Spec FR-007]
- [ ] CHK006 - Описан ли UI баннера "Переподключение..." с позицией, таймингом и поведением при скрытии? [Пробел, Граничный случай §Граничные случаи]

## Ясность требований

- [ ] CHK007 - Количественно ли определено "видное место" для Sign in with Apple с критериями размера/позиционирования? [Ясность, Spec FR-004]
- [ ] CHK008 - Определены ли точные шаги и UI-флоу мастера создания персонажа (сколько экранов, навигация между шагами)? [Ясность, Spec FR-009]
- [ ] CHK009 - Специфицирована ли "визуальная иерархия" для сообщений чата с размерами шрифтов, цветами или отступами между сообщениями DM и игроков? [Неоднозначность, Spec FR-026]
- [ ] CHK010 - Описано ли поведение "разворачиваемых секций" для превью сценария (тап для разворота, автосворачивание других)? [Ясность, Spec FR-017]
- [ ] CHK011 - Определён ли "реальное время" расчёта модификаторов с debounce/throttle поведением для редактора характеристик? [Неоднозначность, Spec FR-011]
- [ ] CHK012 - Перечислены ли конкретные "ключевые события пользователя" для Firebase Analytics помимо четырёх приведённых примеров? [Ясность, Spec FR-054]

## Согласованность требований

- [ ] CHK013 - Согласованы ли требования к компонентам карточек (карточки персонажей, сценариев, комнат) по структуре и стилю? [Согласованность, Spec FR-013, FR-021]
- [ ] CHK014 - Согласованы ли требования к состояниям загрузки на всех экранах (скелетон vs спиннер vs оверлей)? [Согласованность, Spec FR-006, FR-016, FR-031]
- [ ] CHK015 - Согласованы ли паттерны навигации между всеми флоу список->детали (персонажи, сценарии, комнаты)? [Согласованность]
- [ ] CHK016 - Согласовано ли поведение валидации полей ввода между формами аутентификации и другими текстовыми полями? [Согласованность, Spec FR-005]

## Покрытие сценариев

- [ ] CHK017 - Определены ли UI требования для частичных сбоев загрузки данных (например, список персонажей загрузился, но один персонаж не загрузился)? [Покрытие, Исключительный флоу]
- [ ] CHK018 - Рассмотрены ли сценарии параллельных взаимодействий (например, пользователь печатает во время воспроизведения TTS, бросок кубиков во время отправки сообщения)? [Покрытие, Альтернативный флоу]
- [ ] CHK019 - Определены ли UI требования для отключения/переподключения игрока в середине игры с точки зрения других игроков? [Пробел, Флоу восстановления]
- [ ] CHK020 - Определены ли требования для таймаута генерации сценария (очень долгий ответ AI)? [Пробел, Исключительный флоу]
