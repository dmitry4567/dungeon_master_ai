# Implementation Plan: ElevenLabs TTS Streaming

**Branch**: `004-elevenlabs-tts-streaming` | **Date**: 2026-02-27 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/004-elevenlabs-tts-streaming/spec.md`

## Summary

Интеграция ElevenLabs streaming API для потокового воспроизведения текста сообщений DM в реальном времени. Backend проксирует запросы к ElevenLabs с потоковой передачей, клиент воспроизводит аудио через just_audio с поддержкой фонового режима и обработки прерываний.

## Technical Context

**Language/Version**: Python 3.11+ (backend), Dart 3.x / Flutter 3.x (iOS client)
**Primary Dependencies**:
- Backend: FastAPI, httpx (async streaming), pydantic
- Flutter: just_audio ^0.9.36, audio_session ^0.1.18
**Storage**: N/A (no persistent storage, runtime state only)
**Testing**: pytest (backend), flutter_test (client)
**Target Platform**: iOS 15+, Linux server
**Project Type**: Mobile app + API
**Performance Goals**: First audio chunk in <3s, no audible gaps
**Constraints**: ElevenLabs rate limits, 5000 chars/request max
**Scale/Scope**: Single active playback per user

## Implementation Phases

### Phase 1: Backend TTS Service

**Goal**: Создать streaming endpoint для TTS

**Tasks**:
1. Add ElevenLabs settings to `config.py`
2. Create `schemas/tts.py` with request/response models
3. Create `tts_service.py` with:
   - Text preprocessing (strip markdown, DICE tags)
   - Text chunking for long messages
   - ElevenLabs API streaming
   - Error handling (rate limit, quota)
4. Create `routes/tts.py` with endpoints:
   - `POST /tts/stream` - stream audio
   - `GET /tts/status` - service availability
5. Register router in `main.py`
6. Write unit tests for text preprocessing

**Exit Criteria**:
- `curl POST /tts/stream` returns streaming audio
- Error responses match contract
- Tests pass

### Phase 2: Flutter Audio Infrastructure

**Goal**: Настроить аудио-инфраструктуру

**Tasks**:
1. Add `just_audio`, `audio_session` to pubspec.yaml
2. Add `UIBackgroundModes` to Info.plist
3. Create `tts_state.dart` with freezed model
4. Create `tts_cubit.dart` with:
   - AudioPlayer initialization
   - Audio session configuration
   - Interruption handling
   - Single active playback management
5. Create `tts_api.dart` for API calls
6. Register TTSCubit as singleton in DI
7. Run `build_runner` for code generation

**Exit Criteria**:
- Audio plays from test URL
- Phone call interrupts and resumes audio
- Background playback works

### Phase 3: UI Integration

**Goal**: Интегрировать TTS в UI сообщений

**Tasks**:
1. Create `tts_button.dart` widget:
   - Play/Stop icon states
   - Loading indicator
   - Error state display
2. Modify `message_bubble.dart`:
   - Add TTS button to DM messages only
   - Connect to TTSCubit
3. Handle edge cases:
   - Debounce rapid taps
   - Stop on session exit
   - Clear state on logout

**Exit Criteria**:
- TTS button visible on DM messages
- Play/Stop toggles correctly
- Visual states match playback state

### Phase 4: Error Handling & Polish

**Goal**: Обработка ошибок и полировка

**Tasks**:
1. Implement error UI:
   - Rate limit message with retry hint
   - Quota exceeded message
   - Network error handling
2. Add analytics/logging:
   - TTS usage tracking
   - Error rate monitoring
3. Test on real device:
   - Phone call interruption
   - Siri activation
   - Background/foreground transitions
   - Low connectivity

**Exit Criteria**:
- All edge cases from spec handled
- No crashes on interruptions
- User-friendly error messages

## Risks & Mitigations

| Risk | Impact | Mitigation |
|------|--------|------------|
| ElevenLabs downtime | TTS unavailable | Hide buttons, show message |
| Rate limit exceeded | Temp unavailable | Retry-after header, user message |
| Long text latency | Poor UX | Chunk text, stream early |
| iOS audio rejection | App Store block | Follow Apple guidelines exactly |
